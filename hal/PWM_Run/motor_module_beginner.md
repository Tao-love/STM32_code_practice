# motor 模块基础原理说明

## 1. 这个模块是干什么的
这个 `motor` 模块的作用是控制小车两个轮子的电机。

它主要做两件事：
- 控制电机转动方向
- 控制电机转动速度

在你的工程里：
- 左轮用 `MOTOR1` 表示
- 右轮用 `MOTOR2` 表示

---

## 2. 电机为什么不能直接接 STM32
STM32 的 IO 口电流很小，只能输出控制信号，不能直接带动电机。

所以中间必须加电机驱动板，比如：
- L293N
- L298N

STM32 的作用是：
- 发 PWM 信号
- 发方向控制信号

驱动板的作用是：
- 按 STM32 的控制去给电机供电
- 放大电流，真正带动电机转动

---

## 3. ENA/ENB 和 IN1~IN4 是什么
一个电机一般至少要 3 个控制量：
- 1 个使能端
- 2 个方向端

在你的项目里：
- `ENA` 控制左轮速度
- `ENB` 控制右轮速度
- `IN1`、`IN2` 控制左轮方向
- `IN3`、`IN4` 控制右轮方向

可以简单理解成：
- `ENA/ENB` 决定电机“转多快”
- `IN1~IN4` 决定电机“往哪边转”

---

## 4. PWM 为什么能调速
PWM 不是模拟电压，而是“快速开关”的数字波。

比如固定频率 `1kHz`：
- 高电平时间多，电机平均得到的能量就多，转得更快
- 高电平时间少，电机平均得到的能量就少，转得更慢

所以调速的本质不是直接改电压，而是改占空比。

例如：
- `0%`：基本不转
- `25%`：较慢
- `50%`：中等
- `100%`：接近全速

在这个工程里：
- `TIM2_CH1 -> PA0 -> ENA`
- `TIM2_CH2 -> PA1 -> ENB`

也就是 `TIM2` 的两个 PWM 通道分别控制左右轮速度。

---

## 5. 正转、反转、缓停、急停的原理
以一路电机为例，两个方向脚会有 4 种常见组合：

| INa | INb | 电机状态 |
| --- | --- | --- |
| 1 | 0 | 正转 |
| 0 | 1 | 反转 |
| 0 | 0 | 缓停 |
| 1 | 1 | 急停 |

### 5.1 正转
例如左轮：
- `IN1 = 1`
- `IN2 = 0`

这样驱动板给电机一个固定方向的电流，电机正向转。

### 5.2 反转
例如左轮：
- `IN1 = 0`
- `IN2 = 1`

电流方向反过来，电机就反向转。

### 5.3 缓停
例如左轮：
- `IN1 = 0`
- `IN2 = 0`

这时驱动基本断开，电机靠惯性慢慢停下来。
这就叫缓停。

### 5.4 急停
例如左轮：
- `IN1 = 1`
- `IN2 = 1`

这时电机会进入一种电子刹车状态，通常会比缓停停得更快。
这就叫急停。

---

## 6. 为什么速度和方向要分开写
你现在这个模块里：
- `Motor_SetSpeed()` 只负责调速度
- `Motor_Forward()`、`Motor_Reverse()`、`Motor_SlowStop()`、`Motor_FastStop()` 只负责方向脚

这样分开的好处是：
- 逻辑清楚
- 后续好改
- 不容易把速度和方向混乱写在一起

例如你可以先设速度：

```c
Motor_SetSpeed(MOTOR1, 30);
Motor_SetSpeed(MOTOR2, 30);
```

再决定前进还是后退：

```c
Car_Forward();
```

---

## 7. 为什么初始化时先启动 PWM 再停车
`Motor_Init()` 里现在是：
1. 启动 PWM
2. 设默认速度
3. 让两轮先缓停

这样做的目的是：
- PWM 通道先准备好
- 默认速度先设好
- 但方向脚先别让车动起来

这样上电更安全，不会一启动就乱跑。

---

## 8. 你现在这个模块的特点
你现在的写法不是那种抽象很多层的写法，而是最直接的写法：
- `MOTOR1` 就直接对应左轮
- `MOTOR2` 就直接对应右轮
- 看到 `IN1/IN2/IN3/IN4` 就知道在控制什么

优点是：
- 好懂
- 好改
- 适合你现在这个项目

缺点只是“通用性不强”，但现在不重要。

---

## 9. 一句话理解
这个模块本质上就是：
- 用 PWM 控制轮子转得多快
- 用 `IN1~IN4` 控制轮子往前、往后、缓停、急停

这已经是小车运动控制最核心的一层了。
